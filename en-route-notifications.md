# Route notifications to destinations with filters and subscriptions
{: #en-create-en-topic}

Set up event routing in {{site.data.keyword.en_full}} to deliver notifications to your chosen destinations. Create a topic, add event filters, configure subscriptions, and review the configuration before you save it.
{: shortdesc}

## How event routing works
{: #en-route-how-it-works}

Event routing in {{site.data.keyword.en_short}} follows this flow:

1. Create a topic to organize incoming events.
2. Add event filters to define which events are routed to the topic.
3. Create subscriptions to connect the topic to destinations.
4. Review the topic, event filter, and subscription details.
5. Save the configuration to activate the routing flow.

## Before you begin
{: #en-route-prereqs}

Before you start, ensure that you have the following:

* An {{site.data.keyword.cloud_notm}} account
* An {{site.data.keyword.en_short}} service instance
* Appropriate IAM permissions to create topics, filters, and subscriptions
